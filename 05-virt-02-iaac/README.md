# **Задача 2**
1. Убедитесь, что у вас есть ssh ключ в ОС или создайте его с помощью команды  
2. Создайте виртуальную машину Virtualbox с помощью Vagrant и Vagrantfile в директории src.  
3. Зайдите внутрь ВМ и убедитесь, что Docker установлен с помощью команды:
`docker version && docker compose version`  
4. Если Vagrant выдаёт ошибку (блокировка трафика):
```
URL: ["https://vagrantcloud.com/bento/ubuntu-20.04"]     
Error: The requested URL returned error: 404:
```
Выполните следующие действия:

`Используйте зеркало файл-образ "bento/ubuntu-24.04".`

---

## **Решение**

В ходе выполнения задания я столкнулся со следующими проблемами:
1. Так как сценарий использования такой: Proxmox (Хост) -> VM Debian (Рабочая станция для заданий) -> Ubuntu (VirtualBox),  
то входе выполнения задания потребовалось произвести донастройку хоста `echo "options kvm_amd nested=1" > /etc/modprobe.d/kvm.conf`  
и внести изменения в рабочую станцию:
```bash
sudo rmmod kvm_amd kvm
sudo systemctl mask kvm
```
Данное решение позволило запустить VirtialBox внутри уже эмулируемой ОС.  
2. Репозитории vagrant заблокированы на территории РФ. Решение было предоставлено в задание - использование зеркала. Это отражено  
в загруженном vagrantfile;  
3. Также для себя было выполненое небольше задание, в виде добавления открытого ssh ключа внутрь машины. У меня это выполнено  
передачей открытой части в явном виде в vagrantfile. Это решение не безопасно, лучше делать через копирования файла. Но для  
тестогово задания - решил не усложнять.  

Vagrant показался мне интересным решением для работы с CLI-реализацией развертывания ВМ.
